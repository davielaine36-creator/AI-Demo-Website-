# Lane Industries — Client Handoff Checklist

Use this to finish a project cleanly and leave the client feeling confident and
taken care of. A strong handoff is where referrals and repeat work come from.

> Goal: the client understands exactly what they have, how to use it, and how to
> get more help — and Lane Industries is paid in full and set up for the next
> opportunity.

---

## 1. Confirm what was built
- [ ] Write a short, plain-English summary of what was delivered.
- [ ] List each deliverable and where it lives (link/location).
- [ ] Confirm it matches the proposal's Scope/Deliverables.

## 2. Show how to use it
- [ ] Simple written instructions for the everyday tasks (add a lead, send a
      follow-up, check status, etc.).
- [ ] Note anything the client should NOT change without asking.
- [ ] Answer their "but what if…" questions before they arise.

## 3. Return logins / access
- [ ] Hand back or confirm the client owns all logins and accounts.
- [ ] Remove or downgrade any temporary access we were given.
- [ ] Confirm the client controls their own domain, hosting, and tool accounts.
- [ ] Delete any credentials we no longer need to hold.

## 4. Loom walkthrough
- [ ] Record a short Loom (5–10 min) walking through the system live.
- [ ] Cover the main daily actions and where to click.
- [ ] Send the link and note it's theirs to keep and re-watch.

## 5. Maintenance options
- [ ] Offer the Monthly Support / Maintenance plan (see `offer-sheet.md`).
- [ ] Explain what's covered vs. what would be a new project.
- [ ] Make it easy to say yes — but never pushy.

## 6. Final invoice
- [ ] Send the final invoice for the remaining balance.
- [ ] Confirm payment received before considering the project fully closed.
- [ ] Send a simple "thank you / all set" note once paid.

## 7. Testimonial / case study request
- [ ] Once they're happy, ask for a short testimonial (2–3 sentences is plenty).
- [ ] Ask permission to describe the project as a case study.
- [ ] Only use real, approved outcomes — never invented results or numbers.
- [ ] If there are real metrics later, ask if you can add them.

## 8. Follow-up reminder
- [ ] Set a reminder to check in **2–4 weeks** after handoff.
- [ ] Ask how it's going and whether anything small needs adjusting.
- [ ] Set a longer reminder (**60–90 days**) for a bigger check-in + any phase 2.
- [ ] Ask again for referrals ("know anyone else who could use this?").

---

### Handoff message (copy/paste)
> "You're all set! Here's a quick summary of what we built, simple instructions,
> and a short walkthrough video: [links]. Everything's in your control now. I'll
> check in with you in a couple weeks to make sure it's running smoothly — and if
> you ever want ongoing tweaks and support, I've got a simple monthly option.
> It's been a pleasure building this for [Business]."
