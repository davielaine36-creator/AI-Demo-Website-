# Lane Industries — Project Start Checklist

Use this the moment a lead says "yes." It keeps kickoff clean and makes sure you
never start building before the basics (and the deposit) are handled.

> Rule: **no build starts before the offer, price, deposit, and needed info are
> confirmed.** This protects your time and sets a professional tone.

---

## 1. Confirm the offer
- [ ] Confirm exactly which starter build they said yes to.
- [ ] Confirm the goals in one line ("capture leads + draft follow-ups").
- [ ] Re-state the scope so you're both picturing the same thing.

## 2. Confirm the price
- [ ] Confirm the total price for the agreed scope.
- [ ] Confirm deposit amount and balance.
- [ ] Confirm payment method and any monthly support add-on.

## 3. Collect business info
- [ ] Business name, owner name, best contact + method.
- [ ] What the business does, who the customers are.
- [ ] Services/products, service area or location.
- [ ] (Point them to the short or full intake form to capture this cleanly.)

## 4. Collect brand / assets
- [ ] Logo (any format they have) — optional, not a blocker.
- [ ] Brand colors / fonts if they have them.
- [ ] Photos or examples they like — optional.
- [ ] If they have none, note it — we can keep it clean and simple.

## 5. Collect website / domain / platform info (only if the project needs it)
- [ ] Do they have a domain? Where is it registered?
- [ ] Existing website + platform (Wix, Squarespace, Shopify, WordPress, none)?
- [ ] What's working / not working on the current site?
- [ ] Who controls the domain/DNS (for go-live later)?

## 6. Collect tool access — ONLY when necessary
- [ ] Only request access to a tool/account when the build actually requires it.
- [ ] Ask for the least access needed to do the job.
- [ ] Prefer invites/roles over shared passwords where possible.
- [ ] Never collect secrets "just in case" — see `scope-boundaries.md`.
- [ ] Note what access was granted and plan to return/revoke it at handoff.

## 7. Confirm communication method
- [ ] Agree on one main channel (email, text, WhatsApp, etc.).
- [ ] Agree on one main point of contact on their side.
- [ ] Set expectations: response times and how updates will be shared.

## 8. Confirm timeline
- [ ] Confirm start date (after deposit + info received).
- [ ] Confirm estimated delivery window.
- [ ] Note any hard deadlines and whether they're realistic.

## 9. Confirm what success means
- [ ] Write down, in their words, what "this worked" looks like in 30–60 days.
- [ ] Keep it to 1–3 concrete outcomes you can actually point to.

## 10. Send contract / agreement
- [ ] Send the simple agreement / approved proposal (see `proposal-template.md`).
- [ ] Get typed sign-off or an "Approved" reply in writing.

## 11. Collect deposit
- [ ] Send the deposit invoice.
- [ ] Confirm deposit received. **Do not start before this clears.**

## 12. Start build
- [ ] Create a simple project folder/notes doc for this client.
- [ ] Log the project in your tracker (dogfood your own dashboard).
- [ ] Kick off the build and send a quick "we're started" message.

---

### Quick kickoff message (copy/paste)
> "Great — we're all set! I've got everything I need to start. I'll [begin the
> build / send a first draft] by [date], and I'll keep you posted at each step.
> If anything comes up on your end, just message me here."
